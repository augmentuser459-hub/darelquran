# ميزة الخزائن - Treasury Feature

## نظرة عامة
صفحة الخزائن هي ميزة جديدة لتتبع المدفوعات حسب العملة. كل دولة لها خزنة خاصة بعملتها، وكل مدفوعات الطلاب من نفس الدولة تُسجل في خزنة عملتهم.

## الوظائف الرئيسية

### 1. عرض الخزائن
- **8 خزائن** - واحدة لكل دولة/عملة:
  - 🇪🇬 مصر - الجنيه المصري (ج.م)
  - 🇸🇦 السعودية - الريال السعودي (ر.س)
  - 🇦🇪 الإمارات - الدرهم الإماراتي (د.إ)
  - 🇰🇼 الكويت - الدينار الكويتي (د.ك)
  - 🇶🇦 قطر - الريال القطري (ر.ق)
  - 🇧🇭 البحرين - الدينار البحريني (د.ب)
  - 🇴🇲 عمان - الريال العماني (ر.ع)
  - 🇯🇴 الأردن - الدينار الأردني (د.أ)

### 2. معلومات كل خزنة
كل خزنة تعرض:
- **الرصيد الإجمالي**: مجموع كل المدفوعات بهذه العملة
- **مدفوعات اليوم**: المبلغ المدفوع اليوم
- **مدفوعات الأسبوع**: المبلغ المدفوع خلال آخر 7 أيام
- **مدفوعات الشهر**: المبلغ المدفوع في الشهر الحالي
- **عدد المدفوعات**: إجمالي عدد المعاملات

### 3. الإحصائيات العامة
في أعلى الصفحة:
- عدد الخزائن النشطة
- إجمالي عدد المدفوعات
- عدد مدفوعات اليوم
- عدد مدفوعات الأسبوع

### 4. جدول المدفوعات
- عرض تفصيلي لكل المدفوعات
- إمكانية التصفية حسب العملة
- معلومات كل دفعة:
  - رقم الدفعة
  - اسم الطالب
  - الدولة
  - المبلغ
  - العملة
  - التاريخ
  - طريقة الدفع

## كيفية العمل

### آلية حساب الأرصدة
```javascript
// يتم جلب كل المدفوعات المكتملة
payments = SELECT * FROM payments WHERE status = 'completed'

// لكل دولة، يتم حساب:
for each country:
  treasury[country] = {
    total: SUM(payments WHERE currency_code = country.currency_code),
    today: SUM(payments WHERE date = today AND currency = country.currency),
    week: SUM(payments WHERE date >= last_7_days AND currency = country.currency),
    month: SUM(payments WHERE month = current_month AND currency = country.currency)
  }
```

### ربط الطالب بالخزنة
```
طالب → دولة → عملة → خزنة
Student → Country → Currency → Treasury

مثال:
طالب مصري دفع 200 ج.م
↓
الدولة: مصر
↓
العملة: EGP (ج.م)
↓
تُسجل في خزنة الجنيه المصري
```

## الدعم الفني

### في حالة عدم ظهور البيانات
1. تحقق من اتصال Supabase
2. تأكد من وجود مدفوعات مكتملة (status = 'completed')
3. تحقق من ربط الطلاب بالدول
4. راجع Console (F12) للأخطاء

### في حالة أرصدة خاطئة
1. تحقق من حقل `currency_code` في جدول payments
2. تأكد من تطابق العملات مع جدول countries
3. راجع حالة المدفوعات (يجب أن تكون completed)
