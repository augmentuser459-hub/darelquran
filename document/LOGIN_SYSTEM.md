# نظام تسجيل الدخول - Login System

## نظرة عامة
نظام تسجيل دخول بسيط وآمن لحماية لوحة التحكم من الوصول غير المصرح به.

## بيانات الدخول

### المستخدم الافتراضي
```
اسم المستخدم: admin
كلمة المرور: darquran2026
```

## الملفات

### 1. صفحة تسجيل الدخول
**الملف**: `frontend/login.html`

**المميزات**:
- تصميم جذاب ومتناسق مع النظام
- خلفية بنمط إسلامي
- شعار دار القرآن
- آية قرآنية
- حقول إدخال مع أيقونات
- إظهار/إخفاء كلمة المرور
- رسائل خطأ واضحة
- تأثيرات حركية سلسة
- متجاوب مع جميع الشاشات

### 2. ملف المصادقة
**الملف**: `frontend/js/auth.js`

**الوظائف الرئيسية**:

#### `handleLogin(event)`
- التحقق من بيانات الدخول
- إنشاء جلسة عمل (Session)
- تخزين الجلسة في localStorage
- إعادة التوجيه للوحة التحكم

#### `checkAuth()`
- التحقق من وجود جلسة نشطة
- التحقق من صلاحية الجلسة (24 ساعة)
- إزالة الجلسات المنتهية

#### `requireAuth()`
- حماية الصفحات من الوصول غير المصرح
- إعادة التوجيه لصفحة تسجيل الدخول

#### `logout()`
- إنهاء الجلسة
- حذف بيانات الجلسة
- إعادة التوجيه لصفحة تسجيل الدخول

#### `togglePasswordVisibility()`
- إظهار/إخفاء كلمة المرور

## آلية العمل

### 1. تسجيل الدخول
```
المستخدم يدخل البيانات
    ↓
التحقق من صحة البيانات
    ↓
إنشاء جلسة عمل (Session)
    ↓
تخزين في localStorage
    ↓
إعادة التوجيه للوحة التحكم
```

### 2. الجلسة (Session)
```javascript
{
    username: "admin",
    loginTime: 1234567890,
    expiry: 1234567890 + (24 * 60 * 60 * 1000) // 24 hours
}
```

### 3. حماية الصفحات
```javascript
// في بداية كل صفحة محمية
<script src="./js/auth.js"></script>
<script>
    if (!requireAuth()) {
        // سيتم إعادة التوجيه تلقائياً
    }
</script>
```

## حماية الصفحات

### الصفحات المحمية حالياً
- ✅ `index.html` - لوحة التحكم

### لحماية صفحة جديدة
أضف هذا الكود قبل السكريبتات الأخرى:

```html
<!-- Auth Check -->
<script src="../js/auth.js"></script>
```

## زر تسجيل الخروج

### إضافة زر تسجيل الخروج
```html

    <i class="fas fa-sign-out-alt"></i>

```

## مدة الجلسة

### المدة الافتراضية
- **24 ساعة** من وقت تسجيل الدخول

### تغيير المدة
في ملف `auth.js`:
```javascript
const SESSION_DURATION = 24 * 60 * 60 * 1000; // 24 hours

// أمثلة:
// 1 hour: 1 * 60 * 60 * 1000
// 12 hours: 12 * 60 * 60 * 1000
// 7 days: 7 * 24 * 60 * 60 * 1000
```

## الأمان

### التخزين
- يتم تخزين الجلسة في `localStorage`
- لا يتم تخزين كلمة المرور
- يتم التحقق من صلاحية الجلسة في كل صفحة

### نقاط القوة
✅ حماية من الوصول غير المصرح
✅ انتهاء تلقائي للجلسة
✅ عدم تخزين كلمات المرور
✅ رسائل خطأ واضحة

### نقاط الضعف (للعلم)
⚠️ التخزين في localStorage (يمكن الوصول إليه من JavaScript)
⚠️ لا يوجد تشفير للجلسة
⚠️ بيانات الدخول ثابتة في الكود

### تحسينات مستقبلية مقترحة
1. **قاعدة بيانات للمستخدمين**
   - تخزين المستخدمين في Supabase
   - تشفير كلمات المرور (bcrypt)
   - إدارة متعددة للمستخدمين

2. **JWT Tokens**
   - استخدام JSON Web Tokens
   - تشفير أقوى
   - انتهاء صلاحية تلقائي

3. **Two-Factor Authentication**
   - مصادقة ثنائية
   - رموز OTP
   - إشعارات تسجيل الدخول

4. **Session Management**
   - تتبع الجلسات النشطة
   - إنهاء الجلسات من أجهزة أخرى
   - سجل تسجيلات الدخول

5. **Password Recovery**
   - استعادة كلمة المرور
   - إعادة تعيين عبر البريد
   - أسئلة الأمان

## استخدام النظام

### تسجيل الدخول
1. افتح `login.html`
2. أدخل اسم المستخدم: `admin`
3. أدخل كلمة المرور: `darquran2026`
4. اضغط "تسجيل الدخول"

### تسجيل الخروج
1. اضغط زر "تسجيل الخروج" في أي صفحة
2. سيتم إنهاء الجلسة تلقائياً
3. إعادة التوجيه لصفحة تسجيل الدخول

### الوصول المباشر
- إذا حاولت الوصول لصفحة محمية بدون تسجيل دخول
- سيتم إعادة توجيهك تلقائياً لصفحة تسجيل الدخول

## تغيير بيانات الدخول

### في ملف `auth.js`
```javascript
// السطر 4-5
const VALID_USERNAME = 'admin';
const VALID_PASSWORD = 'darquran2026';

// غيّر إلى:
const VALID_USERNAME = 'your_username';
const VALID_PASSWORD = 'your_password';
```

⚠️ **تحذير**: لا تنشر بيانات الدخول على GitHub أو أي مكان عام!

## استكشاف الأخطاء

### لا يمكن تسجيل الدخول
1. تحقق من اسم المستخدم وكلمة المرور
2. افتح Console (F12) وتحقق من الأخطاء
3. تأكد من تحميل ملف `auth.js`

### يتم تسجيل الخروج تلقائياً
1. تحقق من مدة الجلسة (24 ساعة افتراضياً)
2. تحقق من localStorage في المتصفح
3. تأكد من عدم حذف localStorage

### لا يعمل زر تسجيل الخروج
1. تحقق من تحميل ملف `auth.js`
2. تحقق من Console للأخطاء
3. تأكد من استدعاء دالة `logout()`
