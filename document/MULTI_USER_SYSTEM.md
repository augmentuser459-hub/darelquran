# نظام المستخدمين المتعدد

## نظرة عامة
تم إضافة نظام صلاحيات متعدد المستويات يدعم نوعين من المستخدمين:
- **المدير (Admin)**: صلاحيات كاملة لجميع الصفحات
- **المستخدم العادي (User)**: صلاحيات محدودة مع حظر بعض الصفحات

## بيانات تسجيل الدخول

### المدير
- **اسم المستخدم**: `admin`
- **كلمة المرور**: `darquran2026`
- **الصلاحيات**: الوصول لجميع الصفحات

### المستخدم العادي
- **اسم المستخدم**: `user`
- **كلمة المرور**: `user123`
- **الصلاحيات**: الوصول لجميع الصفحات ما عدا:
  - رواتب المحفظين (`teacher-salaries.html`)
  - الخزائن (`treasury.html`)
  - التقارير (`reports.html`)

## الصفحات المحظورة على المستخدم العادي

1. **رواتب المحفظين** (`/pages/teacher-salaries.html`)
   - تحتوي على معلومات مالية حساسة
   - خاصة بالمدير فقط

2. **الخزائن** (`/pages/treasury.html`)
   - إدارة الخزائن والحسابات المالية
   - خاصة بالمدير فقط

3. **التقارير** (`/pages/reports.html`)
   - تقارير مالية وإحصائية شاملة
   - خاصة بالمدير فقط

## آلية عمل النظام

### 1. تسجيل الدخول
- يتم التحقق من بيانات المستخدم في ملف `auth.js`
- يتم إنشاء جلسة (session) تحتوي على:
  - اسم المستخدم
  - الدور (admin/user)
  - اسم العرض
  - وقت تسجيل الدخول
  - وقت انتهاء الجلسة (24 ساعة)

### 2. حماية الصفحات
يتم حماية الصفحات على مستويين:

#### المستوى الأول: Auth Guard
- ملف `auth-guard.js` يعمل قبل تحميل أي صفحة
- يتحقق من وجود جلسة صالحة
- يتحقق من صلاحية الوصول للصفحة الحالية
- يعيد التوجيه للصفحة الرئيسية إذا كانت الصفحة محظورة

#### المستوى الثاني: Auth Module
- ملف `auth.js` يوفر دوال للتحقق من الصلاحيات
- دالة `checkAuth()`: التحقق من الجلسة
- دالة `checkPageAccess()`: التحقق من صلاحية الوصول
- دالة `requireAuth()`: حماية الصفحة وإظهار رسالة خطأ

### 3. إخفاء القوائم
- في الصفحة الرئيسية، يتم إخفاء الروابط المحظورة من القائمة الجانبية
- يتم ذلك باستخدام خاصية `data-restricted="true"`
- كود JavaScript يخفي هذه العناصر للمستخدم العادي

## الملفات المعدلة

### 1. `frontend/js/auth.js`
- إضافة قاعدة بيانات المستخدمين
- تحديث دالة تسجيل الدخول لدعم المستخدمين المتعددين
- إضافة دوال فحص الصلاحيات
- إضافة قائمة الصفحات المحظورة

### 2. `frontend/js/auth-guard.js`
- إضافة فحص الصلاحيات قبل تحميل الصفحة
- إعادة التوجيه للصفحة الرئيسية عند محاولة الوصول لصفحة محظورة
- إضافة رسالة تنبيه عند الحظر

### 3. `frontend/index.html`
- إضافة خاصية `data-restricted="true"` للعناصر المحظورة
- إضافة كود JavaScript لإخفاء العناصر المحظورة
- إضافة رسالة تنبيه عند إعادة التوجيه من صفحة محظورة

### 4. `supabase/create-users-table.sql`
- سكريبت SQL لإنشاء جدول المستخدمين (للاستخدام المستقبلي)
- إضافة المستخدمين الافتراضيين
- إعداد سياسات RLS

## كيفية الاختبار

### اختبار المدير
1. سجل الدخول باسم المستخدم `admin` وكلمة المرور `darquran2026`
2. تحقق من ظهور جميع الروابط في القائمة الجانبية
3. جرب الوصول لجميع الصفحات (يجب أن تعمل جميعها)

### اختبار المستخدم العادي
1. سجل الدخول باسم المستخدم `user` وكلمة المرور `user123`
2. تحقق من اختفاء الروابط المحظورة من القائمة الجانبية
3. جرب الوصول المباشر لصفحة محظورة (مثل `/pages/reports.html`)
4. يجب أن يتم إعادة توجيهك للصفحة الرئيسية مع رسالة خطأ

## إضافة مستخدمين جدد

لإضافة مستخدم جديد، قم بتعديل ملف `frontend/js/auth.js`:

```javascript
const USERS = {
    'admin': {
        password: 'darquran2026',
        role: 'admin',
        name: 'المدير'
    },
    'user': {
        password: 'user123',
        role: 'user',
        name: 'مستخدم'
    },
    // أضف المستخدم الجديد هنا
    'username': {
        password: 'password',
        role: 'user', // أو 'admin'
        name: 'اسم العرض'
    }
};
```

## الأمان

### الحماية الحالية
- التحقق من الجلسة قبل تحميل كل صفحة
- فحص الصلاحيات على مستوى العميل
- إخفاء الروابط المحظورة من الواجهة

### ملاحظات أمنية
⚠️ **مهم**: النظام الحالي يعتمد على التحقق من الصلاحيات على مستوى العميل (Frontend) فقط.

للحصول على أمان أفضل في بيئة الإنتاج:
1. نقل التحقق من الصلاحيات إلى الخادم (Backend)
2. تشفير كلمات المرور في قاعدة البيانات
3. استخدام JWT tokens بدلاً من localStorage
4. إضافة حماية CSRF
5. تفعيل HTTPS

## التطوير المستقبلي

### مقترحات للتحسين
1. **قاعدة بيانات المستخدمين**
   - نقل المستخدمين من الكود إلى قاعدة البيانات
   - استخدام جدول `users` في Supabase

2. **صلاحيات متقدمة**
   - إضافة أدوار أكثر (مثل: محاسب، مشرف، إلخ)
   - صلاحيات على مستوى الإجراءات (قراءة، كتابة، حذف)

3. **إدارة المستخدمين**
   - صفحة لإدارة المستخدمين (إضافة، تعديل، حذف)
   - تغيير كلمة المرور
   - سجل نشاط المستخدمين

4. **الأمان**
   - تشفير كلمات المرور
   - تسجيل محاولات الدخول الفاشلة
   - قفل الحساب بعد عدة محاولات فاشلة
   - إعادة تعيين كلمة المرور

## الدعم الفني

إذا واجهت أي مشاكل:
1. تحقق من console المتصفح للأخطاء
2. تأكد من صحة بيانات تسجيل الدخول
3. امسح cache المتصفح وأعد المحاولة
4. تحقق من صلاحيات المستخدم في ملف `auth.js`
