## خطوات التطبيق على قاعدة البيانات

### إذا كنت تستخدم Supabase:

1. افتح لوحة تحكم Supabase الخاصة بك
2. اذهب إلى **SQL Editor**
3. افتح الملف `supabase/remove-palestine-lebanon.sql`
4. انسخ محتوى الملف والصقه في محرر SQL
5. اضغط على **Run** لتنفيذ الأوامر

### التحقق من التغييرات:

بعد تنفيذ الأوامر، يمكنك التحقق من الدول المتبقية بتشغيل الاستعلام التالي:

```sql
SELECT name_ar, name_en, display_order 
FROM countries 
ORDER BY display_order;
```

## ملاحظات مهمة

### 1. حذف البيانات المرتبطة
عند حذف دولة من جدول `countries`، سيتم تلقائياً:
- حذف جميع خطط التسعير المرتبطة بهذه الدولة (بسبب `ON DELETE CASCADE`)
- **تحذير:** إذا كان هناك طلاب مسجلين من فلسطين أو لبنان، قد تحتاج إلى:
  - نقلهم إلى دولة أخرى
  - أو حذف بياناتهم (غير مستحسن)

### 2. التحقق من الطلاب المتأثرين
قبل تنفيذ الحذف، يُنصح بالتحقق من وجود طلاب من هذه الدول:

```sql
SELECT 
    s.name as "اسم الطالب",
    c.name_ar as "الدولة"
FROM students s
JOIN countries c ON s.country_id = c.id
WHERE c.name IN ('فلسطين', 'لبنان');
```

### 3. نقل الطلاب إلى دولة أخرى (إذا لزم الأمر)
إذا كان هناك طلاب من فلسطين أو لبنان، يمكنك نقلهم إلى دولة أخرى:

```sql
-- مثال: نقل الطلاب من فلسطين إلى الأردن
UPDATE students 
SET country_id = (SELECT id FROM countries WHERE name = 'الأردن')
WHERE country_id = (SELECT id FROM countries WHERE name = 'فلسطين');

-- مثال: نقل الطلاب من لبنان إلى الأردن
UPDATE students 
SET country_id = (SELECT id FROM countries WHERE name = 'الأردن')
WHERE country_id = (SELECT id FROM countries WHERE name = 'لبنان');
```

## التأثير على النظام

### الواجهة الأمامية
- قائمة اختيار الدول في صفحة إضافة/تعديل الطلاب ستعرض فقط الدول المتبقية
- لن يتمكن المستخدمون من اختيار فلسطين أو لبنان عند إضافة طلاب جدد

### التقارير
- التقارير الحالية لن تتأثر
- الطلاب الموجودين حالياً من فلسطين أو لبنان (إن وجدوا) سيظهرون في التقارير حتى يتم نقلهم أو حذفهم
