# دليل سريع: إدارة خزنة الجنيه المصري

## ✅ ما تم إضافته

### 1. أزرار في بطاقة خزنة الجنيه المصري
- 🟢 زر **إيداع**: لإضافة مبلغ للخزنة
- 🔴 زر **صرف**: لخصم مبلغ من الخزنة

### 2. سجل معاملات الخزنة
- جدول يعرض جميع عمليات الإيداع والصرف
- يظهر رواتب المحفظين المدفوعة
- يظهر دفعات الطلاب المستلمة

### 3. تكامل تلقائي
- ✅ عند دفع راتب محفظ → يُخصم من الخزنة تلقائياً
- ✅ عند استلام دفعة طالب بالجنيه → تُضاف للخزنة تلقائياً

## 🚀 خطوات التشغيل

### الخطوة 1: تنفيذ SQL
1. افتح Supabase Dashboard
2. اذهب إلى SQL Editor
3. نفذ الملفات التالية بالترتيب:

```sql
-- أولاً: جدول التحويلات بين الخزائن
-- من ملف: supabase/create-treasury-transfers-table.sql

-- ثانياً: جدول معاملات الخزنة
-- من ملف: supabase/create-treasury-transactions-table.sql
```

### الخطوة 2: افتح صفحة الخزائن
```
http://localhost:8000/pages/treasury.html
```

### الخطوة 3: جرب الميزات
1. ✅ اضغط زر "إيداع" في بطاقة مصر
2. ✅ اضغط زر "صرف" في بطاقة مصر
3. ✅ سجل راتب محفظ وشاهد الخصم التلقائي

## 📝 أمثلة سريعة

### إيداع 1000 جنيه
```
1. اضغط زر "إيداع" الأخضر
2. المبلغ: 1000
3. البند: إيداع نقدي (اختياري)
4. اضغط "تأكيد الإيداع"
✅ النتيجة: +1000 ج.م في الخزنة
```

### صرف 300 جنيه كهرباء
```
1. اضغط زر "صرف" الأحمر
2. المبلغ: 300
3. البند: كهرباء
4. الوصف: فاتورة مارس
5. اضغط "تأكيد الصرف"
✅ النتيجة: -300 ج.م من الخزنة
```

### دفع راتب محفظ (تلقائي)
```
1. اذهب لصفحة رواتب المحفظين
2. سجل راتب جديد
3. حدد تاريخ الدفع
4. احفظ
✅ النتيجة: يُخصم الراتب تلقائياً من خزنة الجنيه
```

## 📊 كيف يُحسب الرصيد

```
الرصيد = دفعات الطلاب + الإيداعات - الصرف - الرواتب
```

### مثال:
```
+ دفعات طلاب: 10,000 ج.م
+ إيداع نقدي: 5,000 ج.م
- صرف كهرباء: -300 ج.م
- راتب محفظ: -1,950 ج.م
─────────────────────────
= الرصيد: 12,750 ج.م
```

## 🎯 الميزات الرئيسية

| الميزة | الوصف |
|--------|-------|
| 🟢 إيداع | إضافة مبلغ للخزنة |
| 🔴 صرف | خصم مبلغ من الخزنة |
| 💰 رواتب | خصم تلقائي عند دفع راتب |
| 💳 دفعات | إضافة تلقائية عند استلام دفعة |
| 📋 سجل | عرض جميع المعاملات |
| 🔍 بحث | البحث في المعاملات |

## ⚠️ ملاحظات مهمة

1. ✅ الميزة تعمل فقط مع **خزنة الجنيه المصري**
2. ✅ النظام يتحقق من الرصيد قبل الصرف
3. ✅ جميع العمليات محفوظة ولا يمكن حذفها
4. ✅ كل عملية لها رقم فريد للتتبع

## 🐛 حل المشاكل

### المشكلة: لا تظهر أزرار الإيداع والصرف
**الحل**: تأكد من تنفيذ ملف SQL في Supabase

### المشكلة: خطأ عند الإيداع أو الصرف
**الحل**: تحقق من:
1. تنفيذ ملف `create-treasury-transactions-table.sql`
2. اتصال Supabase يعمل
3. الصفحة محدثة (F5)

### المشكلة: الرصيد غير صحيح
**الحل**: اضغط زر "تحديث" في أعلى الصفحة

## 📞 الدعم

إذا واجهت أي مشكلة:
1. تحقق من Console في المتصفح (F12)
2. راجع ملف التوثيق الكامل: `TREASURY_TRANSACTIONS_FEATURE.md`
3. تأكد من تنفيذ جميع ملفات SQL
